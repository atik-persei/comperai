# 소개
모든 개발자(및 최종 사용자)가 편리하게 헙업할 수 있도록 최선을 다하고 있습니다.  
기초 요소를 문서에 추가함으로써 모든 사람을 위한 웹을 만드는 것을 목표로 합니다.

## 사용 방법
컴퍼레이는 코드를 분석하여 커밋 메시지를 생성하는 AI 웹 사이트입니다.  
파인튜닝 모델을 이용해 각 줄을 분석하고, 컨벤션에 맞춰 메시지를 생성합니다.  
<br />
좋은 결과를 얻기 위해서는 소스코드 혹은 깃 명령어 결과를 입력하는게 좋습니다.
반환받을 메시지의 언어, 수행한 내용의 커밋 타입, 코드를 작성하고 검색합니다.  

### 코드 입력
소스코드를 그대로 입력하여 깃 커밋 메시지를 반환 받을 수 있습니다.  
파일 이름이 들어있으면 더욱 자세한 메시지 출력이 가능합니다.

###### /app/page.tsx
```tsx
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```




### 명령어 입력
`git diff` 명령어 결과를 입력 데이터로 사용할 수도 있습니다.  
커밋, 브랜치 혹은 레포지토리와 작업 디렉토리 사이의 차이점을 보여줍니다.  
커밋 이력이 존재하지 않는다면 단일 파일의 결과는 보여지지 않을 수 있습니다.  

명령어 결과의 자세한 내용은 [깃 문서](https://git-scm.com/docs/git-diff)를 참고하시기 바랍니다.  

###### Terminal
```ts
git diff /app/page.tsx

diff --git a/app/page.tsx b/app/page.tsx
index ce6b2c7..eaeb973 100644
--- a/app/page.tsx
+++ b/app/page.tsx
@@ -1,6 +1,6 @@
  export default function Page() {
+   return <h1>Test, Next.js!</h1>
-   return <h1>Hello, Next.js!</h1>
  }
```

### 결과 출력
생성된 메시지는 [커밋 컨벤션](/docs/convention/commit-convention)에 맞춰 작성된 결과가 아래의 형태로 구성됩니다.  

###### Terminal
```js
<커밋 타입 이모티콘> <커밋 타입>: <커밋 제목>

<커밋 본문>
```

<br />
출력된 메시지는 생성형 AI가 만들어낸 문장으로 맹신해서는 안됩니다.  
잘못된 문법, 용어, 내용이 있을 수 있습니다. 
확인 후 사용하시는걸 권장드립니다.
